# Project Charter Generation Playbook

## Purpose

Transform validated project discovery information into a structured Project Charter.

The TPM AI Agent must not generate a Project Charter until sufficient information has been collected and validated.

---

# Entry Criteria

The Project Charter generation process can start only when the following information is available:

## Business Context

- Business problem identified.
- Expected business outcome understood.
- Strategic importance clarified.

## Scope

- Major deliverables identified.
- In Scope items defined.
- Out of Scope items identified.

## Stakeholders

- Executive Sponsor identified.
- Key teams impacted identified.
- Decision makers identified.

## Success Criteria

- Expected outcomes defined.
- Success metrics identified.

## Constraints and Dependencies

- Major constraints documented.
- Critical dependencies identified.

---

# Decision Gate

Before generating the Project Charter, ask:

"Do we have enough information to create a useful Project Charter?"

If information is missing:

- Identify missing information.
- Ask targeted questions.
- Do not generate assumptions.

If information is sufficient:

- Confirm understanding with the user.
- Generate the Project Charter.

---

# Generation Guidelines

When creating the Project Charter:

- Focus on business value.
- Use clear executive language.
- Avoid unnecessary technical details.
- Highlight risks and dependencies.
- Keep the document concise and actionable.

---

# Review Process

After generating the Project Charter:

1. Present the draft to the user.
2. Identify assumptions made.
3. Request validation.
4. Incorporate feedback.
5. Produce the final version.
