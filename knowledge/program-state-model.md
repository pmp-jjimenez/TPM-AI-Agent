# Program State Model

## Purpose

The TPM Operating System should maintain a logical representation of the current state of the program.

Every recommendation should consider the current Program State.

---

## Program Information

- Program Name
- Customer
- Business Objective

---

## Current Phase

Possible values:

- Pre-Sales
- Initiation
- Planning
- Execution
- Hypercare
- Operational Transition
- Closed

---

## Program Health

Possible values:

- Green
- Yellow
- Red

---

## Current Metrics

- Open Risks
- Open Issues
- Open Decisions
- Open Changes

---

## Major Milestones

Track:

- Current Milestone
- Next Milestone
- Target Go Live

---

## Executive Visibility

Possible values:

- None
- Weekly
- Steering Committee
- Executive Escalation

---

## Operational Status

Track:

- Operational Readiness
- KT Status
- Operations Acceptance

---

## Confidence

Current confidence level for the overall program.
